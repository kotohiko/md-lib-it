## 1.1   使用计算机网络

计算机网络这个概念，目前学术界并没有一个统一的定义。但是给出一个相对专业、废话较少、要点到位的定义还是不难的。一个简练的定义，可以迅速对这个概念有一个整体的认识，也可以验证自己是否真的理解这个概念。

以下是整理自国内博客、视频网站等互联网平台对计算机网络的一个，我认为还不错的定义。

> **计算机网络（Computer networking）**是一个将众多分散的、自治的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统。

当然，T 教授对于计算机网络的定义也是 OK 的，简练一些，更口语化。

## 1.3   网络技术：从局部到全球

### 1.3.2   局域网

**局域网（Local Area Network, LAN）**是一种私有网络。

所谓**接入点（Access Point, AP）**、**无线路由器（wireless router）**或者**基站（base station）**，虽然这三者具体上不是同一个概念，但是在某种程度上承担着共同的职责，可以理解为同一个概念。比如负责中继（relay）无线计算机之间的数据包（我的台式机开启了一项 Web 服务，我的笔记本可以通过 IP 地址直接访问，前提是这两台设备连接在同一个无线网络），或者负责中继无线计算机与 Internet 的数据包（通过家里常见的 WiFi 路由器来上网）。

**无线局域网（WLAN, wireless local area network）**是一种使用无线通信技术（如无线电波）将设备连接到局域网的方式，使设备可以在没有物理网络线缆的情况下相互通信。WLAN 是一个广泛的概念，包含了多种无线通信标准，如 IEEE 802.11 系列标准。

Wi-Fi 就是 WLAN 的一种实现方式，基于 IEEE 802.11 标准。通常情况下，我们所说的 Wi-Fi 其实是指符合 IEEE 802.11 系列标准的无线通信技术。这种技术能够让设备能够在家庭、公司、咖啡馆等环境中使用无线网络上网。二者并不完全等同，注意区分。

Wireless Fidelity（无线保真）这个更是有趣，很多网站（尤其是国内）甚至直接说这是 Wi-Fi 的 “全称”。实际上这个术语是营销用语，用于对 Wi-Fi 的宣传，但它绝不是 Wi-Fi 的正式技术术语，更不是 Wi-Fi 的 “全称”。Wi-Fi 其实是由 “Wi-Fi Alliance”（Wi-Fi 联盟）创建的一个品牌名称，用来指代符合 IEEE 802.11 标准的无线技术。

所谓**以太网（Ethernet）**，说白了就是一种迄今为止最常见的有线 LAN。它并不直接等同于有线 LAN，而是从属的关系。只是比较普遍罢了。

>   【关于「交换」】
>
>   在计算机网络中，「交换（switching）」指的是在同一网络中通过设备（如交换机）转发数据包的过程。
>
>   交换的目的是让数据包能够在源设备和目标设备之间传输，而不必经过不相关的网络路径。交换通常发生在 LAN）中，交换机使用 MAC 地址来决定数据包的转发路径，从而确保信息高效且准确地到达目标设备。

### 1.3.5   广域网

为了理解和记忆，我们可以把任何终端理解为「资源子网」，除去终端外的任何 “中介” 都是**通信子网（communication subnet）**，包括路由器、交换机、传输介质。

一个网络决定该使用哪一条路径的过程被称为**路由算法（routing algorithm）**。每台路由器决定将一个数据包接下来发送到哪里的过程被称为**转发算法（forwarding algorithm）**。

## 8.12   传输层安全

本节讨论「安全的连接」。

早期的 Web 仅仅用来展示静态页面。但是不久后，有些公司就想到了通过 Web 来做金融交易、信用卡购物、在线银行、股票交易等。很明显这些业务对「安全」要求会比较高。

1995 年，Netscape 通信公司（Netscape Communications Corp.）为了响应这种需求，迅速做出了应对，引入了一种被称为 **SSL（Secure Sockets Layer，安全套接字层）**，如今称为 **TSL（Transport Layer Security，传输层安全）**的安全软件包来满足这种需求。如今这个软件包和它的协议已被广泛使用，包括 Chrome、Safari、Firefox 和 Brave。
